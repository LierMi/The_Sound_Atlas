# 🎼 声音星图 · The Sound Atlas

> **一座可以走进去的音乐博物馆——在星海里选一个音乐时空，让 AI 馆长把那个时代的声音、故事与地理，重建成可进入、可聆听、可带走的展厅。**
> An immersive, AI-curated "time-space museum" of music — pick an era among the stars, and an autonomous AI curator rebuilds its sound, stories, and geography into a hall you can walk into, listen to, and take home.

## 一、🎯 它解决什么问题（需求 · 痛点）

**流媒体把音乐变成了「无限货架上的背景音」。** 你能一键听到一切，却越来越难真正*走进*一首歌、一个时代——

- **发现疲劳**：算法把人困在相似推荐里，「听得多、记得少」，惊喜与意义被稀释；
- **情感稀薄**：歌单是冷的列表，承载不了「这首歌之于我」的私人记忆与时代共鸣；
- **文化失真**：一个时代的声音背后有城市、有故事、有人，但播放器只剩封面和歌名。

| 维度 | 现状 |
|---|---|
| **解决谁的需求** | 怀旧人群、乐迷、文化内容消费者、Z 世代「考古」党 |
| **需求有多痛** | 中高频痛点——「想沉浸、想被打动、想有归属感」，现有产品给不了 |
| **需求有多频繁** | 高频且可运营：每解锁一个时代 = 一次回访；节日 / 纪念日 / IP 联名都是天然节点 |
| **用户量级** | 面向 TME 数亿级用户与亿级泛怀旧人群，可由内容持续驱动增长 |

**声音星图把「播放」重新变回「进入」**：用策展和叙事，把音乐还原成一次可以走进去的文化体验。

## 二、🌌 核心概念（你将体验什么）

把一个音乐时代，做成一座**可进入、可聆听、可带走**的展厅：

1. **星海** — 打开是一片可拖拽环视的 3D 宇宙，每颗发光坐标是一个音乐时空。
2. **俯冲进入** — 点亮一颗星，镜头扎进去，时光机启动。
3. **AI 馆长** — 用「馆长手记 / 展品解说 / 原声试听」重建那个时代的声音现场。
4. **跟着音乐去旅行** — 一条 2D 声音漫游路线（红磡 → 狮子山 → 维港 → 旧唱片铺）。
5. **私人回声** — 你写一句记忆，AI 馆长把它写成只属于你的展签。
6. **盖章带走** — 盖过的时代点亮成你「音乐护照」上的专属星座，一键导出海报分享。

```
星海首页 ──▶ 选一个音乐时空 ──▶ 时光机启动 ──▶ 时空展厅 ──▶ 盖章 ──▶ 音乐护照
 (3D 宇宙)     (发光坐标)        (沉浸过渡)    (AI馆长/试听/    (数字纪念票)  (星座+海报裂变)
                                            路线图/私人回声)
```

**三合一形态**：文化时光机（主线骨架）× 音乐庞贝（展馆格式）× 音乐地图（分享出口）。

## 三、💡 创新点

1. **策展即发现层**——算法推荐之外的另一条路：你不是被投喂歌单，而是走进一个被精心策展的时代。
2. **AI 用在最稳的地方**——不赌「给任意记忆配歌」这种易翻车的事；AI 只做两件下限极高的事：讲好一个**预制时代**的故事、把你的**一句记忆**写成「私人回声」。
3. **沉浸式细节即产品**——真 3D 星海 + 复古天文仪器 HUD + 黑胶唱针落槽 + 票券质感护照，刻意反「模板感」，产品本身就像一件展品。
4. **收集闭环天然裂变**——盖章＝数字纪念票（限量序列号）→ 护照点亮星座 → 一键生成带二维码 / 邀请码的海报，分享即拉新。
5. **数据驱动、可规模化**——加一个时代＝加一条数据，无需改组件；18 个时空已规划，2 个完整、16 个待解锁。

## 四、🤖 AI Agent（长程自主任务 · AI 自主构建）

**产品内 · AI 馆长（策展智能体）**
AI 馆长是一个自主的内容智能体：给定一个时空与馆藏曲目，它自主完成「馆长手记 + 逐曲解说 + 私人回声」的策展叙事，并把用户的私人记忆温柔地织进展厅——让每个人的展厅都不一样。**架构上采用「预制策展 + AI 叙事」混合**：主体内容预置、秒开、有下限；AI 只负责实时的个性化「私人回声」，既快又稳，调用失败自动回退静态文案，绝不白屏。

> 用 AI 的方式，做一个关于 AI 的音乐产品。

## 六、📈 商业价值与 ROI

### 方案效用（用户层面）
把「刷歌单」变成「逛展」，提升**停留 / 听歌时长、复访、情感粘性**；护照的收集与分享带来**自传播拉新**。需求真实、痛点中高频、人群量级大（见第一节）。

### ROI（业务层面）

**🟢 收益（收入 / 规模 / 口碑）**
- **订阅转化**：沉浸式试听种草 → 时空会员 / 完整播放，直接拉动付费与 LTV；
- **用户规模**：护照海报 + 邀请码裂变，获客成本极低；
- **口碑品牌**：高质感、有情感的文化体验，强差异化、易破圈传播；
- **B 端收入**：品牌定制馆 / 版权专馆 / 文旅联名馆的授权与共建收入（高毛利）。

**🟠 成本（可控、极轻）**
- **研发人力**：1 人 + AI 编程智能体长程自主协作，从 0 到上线，人力成本极低；
- **AI / Token**：主体内容**预置、零实时调用**；AI 仅用于一句「私人回声」的短请求，单次成本极小；
- **服务器**：Serverless 架构按需计费，闲时近乎零成本；
- **推广**：增长主要靠护照裂变自传播，买量依赖低。

> **ROI 结论**：收益有订阅 + IP 双引擎，成本结构极轻——**漏斗长在产品骨架里，而不是靠烧钱买量。**

## 七、🛡️ 产品壁垒（竞争力层面 · 为什么难复制）

- **内容架构护城河**：「预制策展 + AI 叙事」的混合架构，既规避「AI 乱配」的致命伤，又保证内容质量有下限——纯 AI 生成或纯人工都难同时做到「稳 + 有温度 + 可规模化」。
- **审美与质感壁垒**：在 AI 让「能做」变廉价的时代，「做得有品」才稀缺；3D 星海、复古仪器、唱针、票券护照的整体质感是逐帧打磨出来的，难以被模板化复制。
- **生态闭环壁垒**：完整变现链路（试听 → 完整播放 → 歌单 → 会员 → 数字藏品 → K 歌 / 长音频）**只有在 TME 自有曲库与产品矩阵内才能真正跑通**——这是外部产品抄不走的结构性优势。
- **数据资产飞轮**：用户点亮的时代、写下的记忆，沉淀为高质量的**年代偏好 + 情感标签**，越用越懂用户，反哺推荐与运营。

## 八、🏗️ 技术实现

- **前端**：React 18 + Vite + Tailwind CSS
- **3D**：three.js + @react-three/fiber + drei + postprocessing（GPU 粒子星海、星球 / 轨道环、Bloom、点击俯冲飞入、移动端降级 + 自适应 DPR）
- **AI 馆长**：大模型驱动的策展智能体（Serverless），生成馆长文案与「私人回声」；**失败自动回退静态文案**，下限稳
- **试听**：展品内置 30s 原声试听（iTunes 预览），无后端也能播
- **护照导出**：`html-to-image` 生成分享海报；`qrcode.react` 生成随海报导出的邀请二维码
- **数据驱动**：全部时空数据集中在 `src/data/eras.js`，加展厅 = 加数据
- **部署**：Vercel（Serverless Functions + 自动构建，已连 GitHub 自动部署）
