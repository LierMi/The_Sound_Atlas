# 声音星图 · The Sound Atlas

> **一座可以走进去的音乐博物馆。**
> 在一片星海里选一个音乐时空 → 俯冲进入 → AI 馆长用手记、展品、原声试听重建那个时代 → 跟着音乐去旅行 → 盖章带走，收进你的「音乐护照」。

**作者 / Author：** Lier Mi
**赛事：** TME 腾讯音乐黑客松 2026 · 赛道 A（创新音乐产品）
**状态：** 可运行 Demo（两个完整展厅：港乐黄金时代、千禧中国风）

## 一、这是什么

流媒体让音乐变成了「无限货架上的背景音」——你能听到一切，却越来越难真正**走进**一首歌、一个时代。

**声音星图**把「播放」重新变回「进入」。它不是又一个播放器皮肤，而是一座**用音乐策展的时空博物馆**：

- 打开是一片可拖拽环视的 **3D 星海**，每一颗发光坐标是一个音乐时空（如「1983–1993 港乐黄金时代」）。
- 点亮一颗星、俯冲进入，**AI 馆长**用「手记 / 展品解说 / 原声试听」把那个时代的声音、故事与地理重建出来。
- 每座展厅有一条 **「跟着音乐去旅行」** 的 2D 路线图（从红馆 → 狮子山 → 维港 → 旧唱片铺）。
- 你写下一句自己的记忆，AI 馆长会把它写成**只属于你的「私人回声」**，别在展签上。
- 喜欢就**盖章带走**——盖过的时代会在你的**音乐护照**上点亮成专属星座，一键导出海报分享。

### 体验流程

```
星海首页 ──▶ 选一个音乐时空 ──▶ 时光机启动 ──▶ 时空展厅 ──▶ 盖章 ──▶ 音乐护照
 (3D宇宙)      (发光坐标)        (过渡)      (AI馆长/试听/    (数字纪念票)  (星座+海报分享)
                                            路线图/私人回声)
```

### 三合一形态

| 形态 | 角色 | 作用 |
|---|---|---|
| 文化时光机 | 主线骨架 | 选时空 → 进入 → 重建一个时代 |
| 音乐庞贝 | 展馆格式 | 把时代「封存」成可进入、可播放的展厅 |
| 音乐地图 | 分享出口 | 护照 = 收集 + 裂变传播的载体 |

## 二、创新点

1. **把「音乐发现」做成「博物馆策展」。** 算法推荐之外的另一条路——策展层成为新的发现层：你不是被投喂歌单，而是走进一个被精心策展的时代。
2. **AI 用在最稳的地方。** 不做容易翻车的「给任意私人记忆配歌」，而是让 AI 做两件下限很高的事：① 讲好一个**预制时代**的故事；② 把用户的**一句话记忆**写成「私人回声」。AI 负责叙事与定制，不负责正确性赌博。
3. **沉浸式细节即体验。** 真 3D 星海 + 复古天文仪器 HUD + 黑胶唱针落槽 + 票券质感的护照——刻意反「模板感」，让产品像一件展品本身。
4. **收集闭环天然可传播。** 盖章 = 数字纪念票（限量序列号）→ 音乐护照点亮星座 → 一键生成带二维码/邀请码的海报，分享即裂变。
5. **数据驱动、可规模化。** 加一个时代 = 在 `src/data/eras.js` 加一条数据，无需改组件。18 个时空已规划，2 个完整、16 个待解锁。

## 三、ROI / 商业价值

它的商业故事一句话：**对 TME 而言，这是一台低成本获客漏斗 + 一个可售卖的「时代 IP 容器」。**

### ROI 漏斗（变现长在产品骨架里，而非硬塞广告）

```
进馆(沉浸) → 试听(种草) → 盖章/收藏(承诺) → 护照分享(裂变) → 回流/付费(变现)
  留存↑        转化触点         私域沉淀          拉新 ↓CAC         订阅 / IP 收入
```

| 价值点 | 对 TME 的意义 |
|---|---|
| 沉浸式内容消费 | 把「刷歌单」变「逛展」，拉高听歌时长 → 订阅留存与 LTV |
| 试听种草 | 每件展品都是一次精准的付费/会员转化触点 |
| 强分享性（护照海报） | 怀旧 + 收集 + 晒图，自传播裂变，**获客成本极低** |
| 文化 IP 属性 | 「时代展馆」可被冠名 / 联名 → B 端买单 |
| 情感数据资产 | 年代偏好 + 记忆标签，反哺推荐与广告 |

## 五、为什么是我（Founder–Product Fit）

这个产品卡在一个稀缺的能力交叉点上——**它同时要求策展审美、独立工程实现，与音乐文化叙事，三者缺一不可：**

- **缺策展审美**，展厅就沦为又一个歌单；
- **缺工程能力**，它永远停在 PPT 与 Figma；
- **缺音乐文化的理解**，叙事就空洞、选曲就失真。

我（**Lier Mi**）的背景正好压在这个交叉上：

1. **策展 / 艺术审美** —— 整个产品的质感（复古仪器 HUD、黑胶唱针、票券护照、星海星座）是审美驱动、逐帧打磨出来的，不是模板能堆的。**审美本身就是壁垒**——在 AI 让「能做」变得廉价的时代，「做得有品」才稀缺。
2. **前端工程** —— 3D 星海、交互转场、护照导出、AI serverless，全部独立落地，从概念到可运行 Demo 一个人闭环。
3. **音乐文化叙事** —— 时代主题、选曲、馆长文案由懂音乐与文化的人把关，保证内容有下限、有温度。

加上架构层面的护城河：**「预制策展 + AI 叙事」的混合内容架构**规避了「AI 乱配」的致命伤，内容质量有下限；而真正完整的变现闭环（试听 → 完整播放 → 歌单 → 会员 → 数字藏品）**只有在 TME 自有曲库体内才能跑通**——这既是天花板，也是别人抄不走的护城河。

## 六、技术实现

- **栈：** React 18 + Vite + Tailwind CSS
- **3D：** three.js + @react-three/fiber(v8) + drei + postprocessing（GPU 粒子星海、星球/轨道环、Bloom、点击飞入、移动端降级 + AdaptiveDpr）
- **AI：** 大模型 API（z.ai GLM，Vercel serverless `api/curator`，生成馆长文案与私人回声；**调用失败自动回退静态文案**，Demo 下限稳）
- **试听：** 展品内置 30s 原声试听（iTunes 预览），无后端也能播
- **护照导出：** `html-to-image` 生成分享海报；`qrcode.react` 生成可随海报导出的邀请二维码
- **数据驱动：** 全部时空数据集中在 `src/data/eras.js`，加展厅 = 加数据

### 流程与关键文件

```
HomePage → LoadingPage → HallPage → PassportPage        （另有 PartnerPage 商务合作页）
```

| `src/components/PartnerPage.jsx` | 商务合作页：B 端四类产品 + 合作意向表单 |

更完整的进展与决策记录见 [`项目文档/进展同步.md`](项目文档/进展同步.md)；商业化一页 pitch 见 [`项目文档/商业化要点.md`](项目文档/商业化要点.md)。
